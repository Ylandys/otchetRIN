Лекция 5 - GIT

Шпора под GIT

  • GIT COMMIT

    Закоммитить изменение файла(сохранить и дать описание проделанной работы)

  • GIT BRANCH

    Создать новую ветку

  • GIT CHECKOUT

    Навигация по веткам(типо CD в CMD)

  • GIT MERGE

    Выполнение слияния веток(мердж веток)

  • GIT REBASE

    При ребейзе Git по сути копирует набор коммитов и переносит их в другое место(Релокация)

  • HEAD

    Выбор КОММИТА

Относительные ссылки - мощный инструмент для перемещения веток

  • GIT RESET

    Переносит указатель ветки на переданный коммит.

  • GIT REVERT

    Данная команда создает новый коммит, который отменяет действие одного из предыдущих коммитов. То есть новый коммит появится в истории, а предыдущие коммиты не изменятся.

  • GIT CHERRY-PICK

    Берет переданный коммит и создает в текущей ветке его точную копию.

  • GIT CLONE

    Клонирует локальный репозиторий создавая его копию

  • GIT PULL

    git pull тоже самое что и git fetch + git merge (Забрать из чужого репозитория)

  • GIT PUSH

    Отправка изменений в чистый репозиторий.
